# Adapting Agency OS for Your Business

The default directory is built for a design and brand consultancy. Here's how to adapt it for other service businesses.

## Dev Shop / Software Agency

**Keep:** operations/, business/, content/

**Swap:**
- `design-strategist.md` → `technical-architect.md`
- `brand-architect.md` → `solutions-architect.md`
- `visual-designer.md` → `frontend-engineer.md`
- `copywriter.md` → `technical-writer.md`
- `presentation-builder.md` → `demo-builder.md`
- `asset-producer.md` → `code-reviewer.md`

**Add:**
- `backend-engineer.md`
- `devops-agent.md`
- `qa-tester.md`
- `security-reviewer.md`

**Remove:** brand-systems/, community/ (unless relevant)

## Marketing Agency

**Keep:** business/, operations/, content/

**Expand content/ to:**
- `seo-strategist.md`
- `paid-media-manager.md`
- `email-marketer.md`
- `social-media-manager.md` (per platform if needed)
- `analytics-analyst.md`

**Swap:**
- `design-strategist.md` → `marketing-strategist.md`
- `brand-architect.md` → `campaign-architect.md`

**Add:**
- `media-buyer.md`
- `conversion-optimizer.md`
- `influencer-coordinator.md`

## Content Studio

**Keep:** business/, operations/

**Expand content/ to the core:**
- `editor-in-chief.md`
- `longform-writer.md`
- `shortform-writer.md`
- `video-scriptwriter.md`
- `podcast-producer.md`
- `seo-editor.md`
- `fact-checker.md`

**Swap:**
- `design-strategist.md` → `content-strategist.md`
- `visual-designer.md` → `thumbnail-designer.md`

## Consulting Firm

**Expand strategy/ to the core:**
- `industry-analyst.md`
- `frameworks-specialist.md`
- `workshop-facilitator.md`
- `executive-advisor.md`

**Keep:** business/, operations/

**Simplify creative/ to:**
- `presentation-builder.md`
- `report-writer.md`

## The Pattern

Regardless of agency type, the structure stays the same:

1. **shared-soul.md** — your firm's DNA (always needed)
2. **strategy/** — the thinking layer
3. **creative/** or **production/** — the making layer
4. **content/** — the distribution layer
5. **business/** — the revenue layer
6. **operations/** — the keep-it-running layer

The departments adapt. The architecture doesn't.
